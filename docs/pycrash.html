PyCrash Module Documentaion¶

This is the core documentation of the core PyCrash documentation.

PyCrash - A Run-Time Exception Dumper for Python Applications.

PyCrash allows you to eayly add crash reporting facilities to your python applications. It supports the gneraton of crash reports in both XML and HTML.

class pycrash.PyCrash(**options)[source]¶

This object is a singleon and when enabled will overided utilize the sys.exceptHook.

addCustomRecord(record)[source]¶

Custom records are optional bits of information that can be included in the crash report.

disable()[source]¶

Disable PyCrash to trace raised exceptions

enable()[source]¶

Enable PyCrash to trace raised exceptions

forceDump()[source]¶

forceDump() forces the creation of the crash dump, if an exception is raised. forceDump() method can be very useful when an exception hasn’t reached the top-level, but the user wants the same to make a dump of the application context

isCrashed()[source]¶

Returns 1 (true) if the application has been crashed, otherwise 0

onBegin()[source]¶

User’s defined actions: derived classes can override this method to add custom actions. This method is called at the begin of crash dump construction

onEnd()[source]¶

User’s defined actions: derived classes can override this method to add custom actions. This method is called at the end of crash dump construction

onExceptionRaised(time)[source]¶

User’s defined actions: derived classes can override this method to add custom actions. This method is called every time an uncaught exception is raised: “time” reports the number of uncaught exceptions raised

onExit()[source]¶

User’s defined actions: derived classes can override this method to add custom actions. This method is called at the exit of application

registerDispatcher(dispatcher)[source]¶

This method will register a dispatcher

saveToFile(filename)[source]¶

Save the crash dump file in a given ‘filename’ directory

toXML()[source]¶

Returns the XML rapresentation of the crash dump
